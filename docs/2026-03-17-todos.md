## 4. Free-tier flow & paywall segmentation

This is the biggest growth lever. Currently, all widgets require sign-up + auth token. The conversion rate is ~0.3% (very poor). Letting people use widgets without sign-up, then gating premium features, would dramatically increase engagement and create natural upgrade moments.

### Proposed segmentation (needs design doc before implementation)

**Free (no sign-up required):**
- Basic widget functionality (start a pomodoro, log water, check off habits)
- "Powered by Blocs" branding visible
- Data stored locally (localStorage) — no persistence across devices
- Limited to defaults (no custom presets, no theme changes)

**Pro ($5/mo or $36/yr, requires sign-up):**
- Data saved to cloud (persistent across devices, survives browser clears)
- Unlimited analytics history (weekly/monthly reports)
- Custom pomodoro presets and widget themes
- No branding
- Share progress with friends
- All future widgets

### Key design decisions to make
- [ ] Does the free widget need any sign-up at all, or is it fully anonymous with localStorage?
- [ ] What's the upgrade trigger? Options:
  - User tries to access analytics -> paywall
  - User tries to customize theme/presets -> paywall
  - After X days of local usage, prompt to "save your data" by signing up
  - Data limit: "You have 7 days of history. Upgrade for unlimited."
- [ ] How does the embed URL work for free users? Currently, URLs require a token. Free embeds would need to work without one.
- [ ] Do we keep the dashboard for managing embed links, or can free users get an embed link directly from the widget page?

### Implementation phases
1. **Phase 1:** Make widget pages show functional demos for unauthenticated visitors (use existing dummy components). Add CTAs.
2. **Phase 2:** Build localStorage-based free widgets that work without sign-up. Design the paywall triggers.
3. **Phase 3:** Rework the embed flow so free users can get a working embed link without the dashboard.
